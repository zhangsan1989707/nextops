## 2. 产品背景

传统运维平台通常以菜单、表单、脚本和命令行为核心。用户在完成一次部署、巡检、排障或资源纳管时，需要在多个系统之间切换，包括监控系统、告警系统、CMDB、堡垒机、脚本平台、CI/CD、日志平台和云平台控制台。操作路径长、上下文割裂、标准化程度不足，导致响应慢、协作难、排障依赖个人经验。

NextOps 希望将复杂运维操作收敛到一个现代化 SaaS 平台中，以自然语言和 Slash 指令作为入口，让对话框成为控制台，让 AI 能够理解资产、日志、指标、告警、脚本和变更上下文，帮助用户快速完成部署、巡检、排障、修复建议生成和自动化执行。

## 3. 产品定位

NextOps 是一个面向企业 IT、SRE 与 DevOps 团队的 AIOps 运维平台。平台以 ChatOps 为核心交互方式，整合资产纳管、服务器监控、告警治理、AI 诊断、脚本编排、插件驱动、权限审批和多租户管理能力。

核心目标是让用户通过一句自然语言或一个 Slash 指令完成高频运维动作，并在必要时通过审批、权限、审计和人工确认保证操作安全。

## 4. 产品愿景

让运维从“找系统、查指标、翻日志、写脚本、跑命令”转变为“描述目标、确认方案、自动执行、持续反馈”。

NextOps 的长期愿景包括：

- 将自然语言变成运维入口。
- 将对话框变成统一控制台。
- 将 AI 诊断变成排障助手。
- 将脚本、插件、协议、云资源变成可编排能力。
- 将部署、巡检、排障、告警治理沉淀为可复用的团队知识。

## 5. 目标用户与角色

### 5.1 运维工程师 / SRE

关注服务器状态、部署任务、巡检任务、告警处理、故障定位和修复执行。需要快速掌握资产健康度，并通过脚本、命令、AI 分析提升处理效率。

### 5.2 DevOps 工程师

关注 CI/CD、环境发布、流水线联动、制品包管理、配置变更、回滚和发布审计。需要平台平滑接入现有 DevOps 体系。

### 5.3 研发负责人 / 技术负责人

关注服务稳定性、故障影响范围、性能趋势、成本与资源使用情况。需要查看大盘、趋势、告警分布和修复建议。

### 5.4 平台管理员

关注模型配置、成员管理、团队结构、权限角色、租户隔离和系统集成。

### 5.5 租户管理员

关注本租户内的资产、成员、告警规则、工单审批和操作审计。

## 6. 核心价值

### 6.1 自然语言触发复杂操作

用户可以输入“帮我巡检 10.0.1.21 的 CPU、内存和磁盘，并生成风险报告”，系统自动识别目标资产、操作类型、所需权限和执行步骤。

### 6.2 Slash 指令直达高频操作

用户可通过 `/deploy`、`/check`、`/ssh`、`/restart`、`/logs`、`/alert` 等指令快速触发部署、巡检、排障、日志查询、告警处理和服务器连接。

### 6.3 对话框就是控制台

ChatOps 页面不仅展示 AI 回复，还可以承载任务确认、脚本参数填写、审批状态、执行日志、实时输出、修复方案和结果回写。

### 6.4 一站式服务器纳管

平台内置 Web SSH。用户输入 IP 和端口即可连接服务器，并可通过 Agent 一键部署实现服务器纳管，查看 CPU、内存、磁盘、实时监控、性能图、系统配置和告警规则。

### 6.5 AI 诊断关联日志、指标和数据

AI 可以聚合日志、监控指标、告警记录、资产信息、部署记录和历史工单，主动识别性能趋势，定位可能原因，并生成可执行修复方案。

### 6.6 插件化与多协议驱动

平台通过插件中心连接传统基础设施、云原生环境、DevOps 工具链、私有云、日志系统、监控系统、CMDB、Kubernetes、主机、数据库、中间件和制品仓库。

## 9. 核心业务流程

### 9.1 自然语言运维流程

1. 用户在 ChatOps 输入自然语言需求。
2. 系统识别意图、目标资产、风险等级和所需参数。
3. 系统返回执行计划，展示涉及服务器、脚本、命令、权限和预计影响。
4. 用户确认执行；高风险操作进入工单审批。
5. 系统执行任务并实时展示日志、进度和状态。
6. 任务结束后生成结果摘要、失败原因和后续建议。
7. 操作记录进入审计日志，可被 AI 后续引用。

### 9.2 Slash 指令流程

1. 用户输入 `/` 唤起快捷指令面板。
2. 系统根据权限、场景和历史使用频率展示指令。
3. 用户选择指令并填写参数。
4. 系统校验参数、权限和风险等级。
5. 执行任务并在对话中返回结构化结果。

### 9.3 服务器纳管流程

1. 用户进入服务器管理页面。
2. 点击“新增服务器”。
3. 输入 IP、端口、认证方式和所属租户。
4. 系统尝试 Web SSH 连接。
5. 用户选择一键部署 Agent。
6. Agent 上报主机信息、系统配置和监控指标。
7. 服务器进入资产列表，状态变为已纳管。

### 9.4 AI 诊断流程

1. 告警触发或用户主动发起诊断。
2. 系统关联告警、日志、指标、资产、部署记录和历史事件。
3. AI 分析异常时间段、性能趋势、错误模式和影响范围。
4. 系统输出可能原因、证据链、影响判断和修复方案。
5. 用户选择执行修复、生成工单、转人工或加入知识库。

### 9.5 告警规则配置流程

1. 用户进入服务器详情页。
2. 查看服务器专属告警规则。
3. 用户可指定已有规则或新建规则。
4. 新建规则支持选择指标、阈值、持续时间、通知方式和自动化动作。
5. 保存后规则立即生效，并进入告警中心统一管理。

## 10. 功能需求

## 10.1 仪表盘

### 功能目标

提供全局运维态势总览，让用户快速了解资产、告警、性能、任务和 AI 诊断情况。

### 功能点

- 展示服务器总数、在线数、异常数、未纳管数。
- 展示 CPU、内存、磁盘、网络总体健康度。
- 展示当前告警数量，按严重级别分类。
- 展示近 24 小时告警趋势、故障趋势和恢复趋势。
- 展示最近执行的部署、巡检、脚本和 AI 诊断任务。
- 展示高风险服务器、性能异常服务和待处理工单。
- 支持按租户、团队、环境、标签、时间范围筛选。

### 关键指标

- 平台资产健康度。
- 告警收敛率。
- 平均故障响应时间。
- 平均故障恢复时间。
- AI 诊断采纳率。

## 10.2 ChatOps

### 功能目标

将自然语言、Slash 指令、任务执行和结果反馈统一到一个对话式控制台。

### 功能点

- 支持自然语言输入。
- 支持 Slash 指令。
- 支持上下文追问，例如“把刚才那台机器也加上磁盘巡检”。
- 支持部署、巡检、排障、日志查询、指标查询、服务器连接、脚本执行。
- 支持 AI 返回执行计划。
- 支持用户确认、取消、修改参数。
- 支持实时任务日志输出。
- 支持执行结果结构化展示。
- 支持将诊断结果生成工单、脚本或知识库条目。
- 支持历史会话检索。
- 支持按租户和权限隔离会话数据。

### 示例输入

- “帮我巡检生产环境所有 Web 服务器。”
- “查看 10.0.1.21 最近 30 分钟的 CPU 和 nginx 错误日志。”
- “帮我分析这次数据库连接数飙升的原因。”
- `/ssh 10.0.1.21 --port 22`
- `/deploy order-service --env prod --version 1.8.2`
- `/check prod-web --items cpu,memory,disk`

## 10.3 告警中心

### 功能目标

统一管理告警事件、告警规则、通知策略和告警处理流程。

### 功能点

- 展示告警列表，支持严重级别、状态、来源、租户、服务器、时间筛选。
- 告警状态包括待处理、处理中、已恢复、已忽略、已关闭。
- 支持告警详情页，展示触发规则、指标曲线、日志片段、影响资产和历史相似事件。
- 支持 AI 一键诊断。
- 支持告警认领、转派、备注、升级和关闭。
- 支持告警规则新建、编辑、启用、禁用和克隆。
- 支持通知策略配置，包括邮件、Webhook、企业 IM 和工单。
- 支持告警静默、抑制、聚合和去重。

## 10.4 服务器管理

### 功能目标

实现服务器纳管、监控、连接、配置查看、告警规则绑定和基础运维操作。

### 功能点

- 服务器列表展示 IP、主机名、系统类型、环境、租户、状态、Agent 状态、CPU、内存、磁盘使用率。
- 支持按标签、环境、租户、状态、系统类型筛选。
- 支持新增服务器。
- 支持输入 IP 和端口进行 Web SSH 连接。
- 支持 Agent 一键部署。
- 支持批量部署 Agent。
- 支持服务器详情页。
- 支持查看 CPU、内存、磁盘、网络、进程、端口、负载、文件系统。
- 支持实时监控和性能图。
- 支持查看系统详细配置，包括 OS、内核、CPU 型号、内存容量、磁盘分区、网卡、启动时间。
- 支持绑定专属告警规则。
- 支持指定已有规则。
- 支持新建规则。
- 支持基础操作：重启服务、执行脚本、文件上传下载、查看日志、发起 AI 诊断。

## 10.5 Web SSH

### 功能目标

在浏览器中直接连接服务器，降低终端工具依赖，并与平台权限、审计和 AI 能力结合。

### 功能点

- 支持输入 IP、端口、用户名和认证方式连接。
- 支持密码、密钥、临时凭证和托管凭证。
- 支持终端窗口多标签。
- 支持命令审计和会话回放。
- 支持敏感命令识别与二次确认。
- 支持从服务器列表一键打开 SSH。
- 支持将终端输出发送给 AI 进行诊断。
- 支持断线重连。

## 11. 页面设计要求

### 11.1 整体风格

界面采用现代 SaaS 风格，强调专业、清晰、高效和响应式体验。视觉应避免传统后台系统的沉重感，同时保持运维场景所需的信息密度。

### 11.2 布局

- 左侧为工具栏导航。
- 顶部为全局搜索、租户切换、通知、用户入口。
- 主内容区根据页面类型展示大盘、列表、详情、终端或对话。
- 支持导航折叠。
- 支持桌面端多栏布局。
- 平板端保持核心操作可见。
- 移动端优先保留告警、ChatOps、服务器状态和审批处理。

### 11.3 交互原则

- 高频操作优先通过快捷按钮、Slash 指令和上下文菜单完成。
- 危险操作必须二次确认。
- 高风险操作必须支持审批。
- AI 给出的自动化操作必须先展示计划和风险。
- 执行中的任务必须展示实时状态。
- 失败结果必须给出原因和下一步建议。

## 16. 验收标准

- 用户可以通过 IP 和端口新增服务器并打开 Web SSH。
- 用户可以一键部署 Agent 并在服务器详情页看到基础指标。
- 用户可以在 ChatOps 输入自然语言并得到可执行计划。
- 用户可以通过 Slash 指令触发高频操作。
- 用户可以查看服务器 CPU、内存、磁盘、实时监控、性能图和系统配置。
- 用户可以为服务器指定已有告警规则或新建规则。
- 用户可以在告警中心查看、处理和关闭告警。
- 用户可以发起 AI 诊断，并得到日志、指标、可能原因和修复方案。
- 用户可以在脚本中心创建脚本，并通过快捷指令或 ChatOps 调用。
- 管理员可以配置模型、成员、团队、角色和权限。
- 多租户数据必须隔离，不同租户无法互相查看资产和告警。

## 17. 风险与约束

### 17.1 AI 输出安全风险

AI 可能生成错误命令或高风险修复方案。所有自动化执行必须经过权限校验、风险识别和用户确认，高风险动作进入审批。

### 17.2 服务器凭证安全

服务器连接涉及密码、密钥和 Token。平台必须加密存储凭证，并对访问、使用和导出进行严格限制。

### 17.3 Agent 部署适配风险

不同操作系统、网络环境和安全策略可能影响 Agent 部署成功率。需要提供手动安装脚本、失败诊断和离线部署方案。

### 17.4 多系统集成复杂度

DevOps、私有云、日志、监控和 CMDB 系统差异较大。插件体系需要具备标准接口、错误处理和版本兼容机制。
