# NextOps MVP 研发排期

## 阶段 0：工程地基

目标：本机可运行，CI 可检查。

- 初始化 monorepo。
- 建立 Web、API、Docker Compose。
- 建立 PRD、架构、数据库、API 文档。
- 增加基础 CI 和 Jenkinsfile。

验收：

- `npm run docker:up` 可以启动 Web/API/Postgres/Redis。
- Web 可访问。
- API health 可访问。

## 阶段 1：基础管理台

目标：能看见 NextOps 的产品形态。

- SaaS 左侧导航。
- 仪表盘。
- 服务器列表。
- 服务器详情。
- 告警中心列表。
- 脚本中心列表。

验收：

- 用户能通过界面浏览核心菜单。
- 服务器、告警、脚本使用 API 数据渲染。

## 阶段 2：ChatOps Demo 闭环

目标：一句话触发运维计划。

- ChatOps 输入框。
- Slash 指令提示。
- `/ssh`、`/check`、`/diagnose` mock 执行。
- 返回执行计划、风险和下一步动作。

验收：

- 输入自然语言可得到结构化任务计划。
- 输入 Slash 指令可得到对应动作结果。

## 阶段 3：真实数据与权限

目标：从 mock 切到数据库。

- PostgreSQL 表结构落地。
- 登录与基础 RBAC。
- 租户隔离。
- 服务器 CRUD。
- 审计日志。

验收：

- 管理员、运维、只读角色权限不同。
- 操作记录可审计。

## 阶段 4：Agent 与 Web SSH

目标：具备真实主机接入能力。

- Agent 注册和心跳。
- 指标上报。
- Agent 安装脚本。
- Web SSH 基础连接。
- 命令审计。

验收：

- 一台测试服务器可被纳管。
- 页面可看到真实指标。
- Web SSH 可打开终端。

## 阶段 5：AI 诊断与自动化

目标：AIOps 能力形成闭环。

- 模型管理。
- 日志和指标关联。
- AI 诊断报告。
- 修复方案生成。
- 脚本草稿生成。
- 高风险操作审批。

验收：

- 告警可触发 AI 诊断。
- 诊断报告包含证据链和修复建议。
- 修复执行有确认、审批和审计。
